# ReportGenie — Submission Write-Up

**Track:** Agents for Business
**Project Name:** report-genie
**ADK Version:** 2.0

---

## 1. Problem Statement

Business analysts and operations teams spend hours manually pulling metrics from databases, formatting reports, and emailing summaries to stakeholders. This process is error-prone, slow, and blocks data-driven decision-making. ReportGenie automates the entire pipeline — from data collection to report formatting and stakeholder communication — while keeping a human in the loop before anything is sent.

---

## 2. Solution Architecture

ReportGenie is a **multi-agent ADK 2.0 Workflow** with the following node graph:

```
User Input (plain text)
        │
        ▼
┌──────────────────────┐
│ security_checkpoint  │ ← FunctionNode — PII scrub · injection detect · audit log
└──────────┬───────────┘
           │
      [ok] │   [security_violation]
           ├─────────────────────────┐
           ▼                         ▼
┌───────────────────┐     ┌─────────────────────────┐
│   orchestrator    │     │ security_violation_node │
│   (LlmAgent)      │     └─────────────────────────┘
│  ┌──────────────┐ │
│  │data_collector│ ├─────◄ MCP: query_database
│  │ (AgentTool)  │ │       MCP: read_business_file
│  └──────┬───────┘ │
│         │         │
│  ┌──────▼───────┐ │
│  │ report_gen.  │ ├─────◄ MCP: generate_chart_data
│  │ (AgentTool)  │ │
│  └──────────────┘ │
└──────────┬────────┘
           │
           ▼
┌─────────────────────┐
│ parse_orchestrator  │ ← FunctionNode — JSON extraction from ctx.state
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ hitl_approval_node  │ ← HITL — RequestInput pause for human review
└──────────┬──────────┘
           │
[approved] │ [rejected]
  ┌────────┴────────┐
  ▼                 ▼
┌──────────┐  ┌────────────┐
│send_email│  │ reject_node│
└──────────┘  └────────────┘
```

## 3. Concepts Used

### ✅ ADK Multi-Agent (Workflow + LlmAgents + AgentTool + ctx.state)
- **File:** [`app/agent.py`](app/agent.py)
- A `Workflow` (ADK 2.0 graph API) orchestrates the entire flow.
- Three `LlmAgent` instances: `orchestrator`, `data_collector`, `report_generator`.
- The orchestrator uses `AgentTool(data_collector)` and `AgentTool(report_generator)` to delegate work to specialized sub-agents.
- `ctx.state` is used to pass the sanitized query and the orchestrator's JSON output between nodes via `state={"sanitized_query": ...}` and `output_key="orchestrator_result"`.

### ✅ MCP Server (mcp_server.py + 3 tools + MCPToolset in 2+ agents)
- **File:** [`app/mcp_server.py`](app/mcp_server.py)
- Built with the `FastMCP` class from the `mcp` Python SDK (stdio transport).
- Exposes 3 domain tools:
  - `query_database` — returns mock sales, HR, and customer metrics
  - `read_business_file` — reads CSV/log file content (support tickets, campaigns)
  - `generate_chart_data` — returns ASCII bar charts for visual reporting
- Wired into both `data_collector` and `report_generator` via `McpToolset`.

### ✅ Security Checkpoint (security_checkpoint FunctionNode)
- **File:** [`app/agent.py`](app/agent.py) — `security_checkpoint()` function node
- Sits at the entry point of the workflow, before any LLM calls.
- Full details in Section 4 below.

### ✅ Agents CLI (scaffold + GEMINI.md + Makefile)
- Project scaffolded via: `uvx google-agents-cli scaffold create report-genie --deployment-target agent_runtime --agent adk`
- `GEMINI.md` generated by scaffold — documents agent directory and conventions.
- `Makefile` includes `install`, `playground`, `run`, `test` targets.
- Skills installed: `google-agents-cli-workflow`, `google-agents-cli-adk-code`, `google-agents-cli-scaffold` and 4 others.

## 4. Security Design

| Control | Implementation | Why It Matters |
|---------|---------------|----------------|
| **PII Scrubbing** | Regex redacts emails (`[REDACTED_EMAIL]`), phone numbers (`[REDACTED_PHONE]`), and SSNs (`[REDACTED_SSN]`) before the query reaches any LLM | Prevents personally identifiable data from being embedded in LLM prompts or audit logs |
| **Prompt Injection Detection** | 6 keyword patterns checked: `"ignore previous instructions"`, `"system prompt"`, `"override instructions"`, `"bypass safeguards"`, `"ignore rules"`, `"you are now a"` | Stops adversarial inputs from hijacking agent behaviour |
| **Domain-Specific Rule** | Queries containing `password`, `credential`, `secret_key`, or `api_key` are blocked | Prevents credential exfiltration via the reporting pipeline |
| **Structured Audit Log** | Every request logged as JSON: `{"severity": "INFO|WARNING|CRITICAL", "audit": {...}}` | Full traceability of security decisions for compliance review |
| **Route-based Blocking** | Blocked requests route to `security_violation_node` — no further LLM calls made | Ensures zero LLM compute is spent on malicious inputs |

## 5. MCP Server Design

**File:** [`app/mcp_server.py`](app/mcp_server.py)

| Tool | Purpose | Used By |
|------|---------|---------|
| `query_database(query)` | Queries mock corporate database for sales, HR, or customer KPI data | `data_collector` |
| `read_business_file(file_name)` | Reads business logs/CSVs (support tickets, marketing campaigns) | `data_collector` |
| `generate_chart_data(metric_name)` | Returns ASCII bar charts for visual embedding in reports | `report_generator` |

The MCP server runs as a subprocess via stdio transport using `uv run python -m app.mcp_server`. This means no additional port or installation is required — it starts on-demand when any agent session begins.

## 6. HITL Flow

Human approval is required before any report is dispatched.

| Point | Mechanism | Why |
|-------|-----------|-----|
| After orchestrator generates draft | `RequestInput(interrupt_id="approve_report")` pauses the workflow | Ensures a human reviews content for accuracy before it reaches stakeholders |
| User replies `yes` | Routes to `send_email_node` → confirmation shown | Explicit consent to send |
| User replies `no` (or anything else) | Routes to `reject_node` → cancellation shown | Prevents accidental or incorrect reports from being sent |

This HITL gate is critical for a business reporting context where incorrect data sent to executives can have significant consequences.

## 8. Impact / Value Statement

ReportGenie demonstrates that AI agents can automate the most repetitive parts of business intelligence work — data collection, report formatting, and stakeholder communication — while keeping humans fully in control of what gets sent.

**Who benefits:**
- **Operations & Analytics teams** — save hours of manual report compilation per week
- **Executives & stakeholders** — receive consistent, well-formatted reports on demand
- **IT & Compliance teams** — every request is audited with PII protection built in

**Broader potential:** The architecture is domain-agnostic. Swap the MCP tools for real database connectors, email APIs, or BI systems and the same workflow operates on live enterprise data — with the same security guarantees and human oversight.
