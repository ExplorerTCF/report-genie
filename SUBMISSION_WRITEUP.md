## 1. Problem Statement

Business analysts and operations teams spend hours manually pulling metrics from databases, formatting reports, and emailing summaries to stakeholders. This process is error-prone, slow, and blocks data-driven decision-making. ReportGenie automates the entire pipeline вҖ” from data collection to report formatting and stakeholder communication вҖ” while keeping a human in the loop before anything is sent.

## 3. Concepts Used

### вң… ADK Multi-Agent (Workflow + LlmAgents + AgentTool + ctx.state)
- **File:** [`app/agent.py`](app/agent.py)
- A `Workflow` (ADK 2.0 graph API) orchestrates the entire flow.
- Three `LlmAgent` instances: `orchestrator`, `data_collector`, `report_generator`.
- The orchestrator uses `AgentTool(data_collector)` and `AgentTool(report_generator)` to delegate work to specialized sub-agents.
- `ctx.state` is used to pass the sanitized query and the orchestrator's JSON output between nodes via `state={"sanitized_query": ...}` and `output_key="orchestrator_result"`.

### вң… MCP Server (mcp_server.py + 3 tools + MCPToolset in 2+ agents)
- **File:** [`app/mcp_server.py`](app/mcp_server.py)
- Built with the `FastMCP` class from the `mcp` Python SDK (stdio transport).
- Exposes 3 domain tools:
  - `query_database` вҖ” returns mock sales, HR, and customer metrics
  - `read_business_file` вҖ” reads CSV/log file content (support tickets, campaigns)
  - `generate_chart_data` вҖ” returns ASCII bar charts for visual reporting
- Wired into both `data_collector` and `report_generator` via `McpToolset`.

### вң… Security Checkpoint (security_checkpoint FunctionNode)
- **File:** [`app/agent.py`](app/agent.py) вҖ” `security_checkpoint()` function node
- Sits at the entry point of the workflow, before any LLM calls.
- Full details in Section 4 below.

### вң… Agents CLI (scaffold + GEMINI.md + Makefile)
- Project scaffolded via: `uvx google-agents-cli scaffold create report-genie --deployment-target agent_runtime --agent adk`
- `GEMINI.md` generated by scaffold вҖ” documents agent directory and conventions.
- `Makefile` includes `install`, `playground`, `run`, `test` targets.
- Skills installed: `google-agents-cli-workflow`, `google-agents-cli-adk-code`, `google-agents-cli-scaffold` and 4 others.

## 4. Security Design

| Control | Implementation | Why It Matters |
|---------|---------------|----------------|
| **PII Scrubbing** | Regex redacts emails (`[REDACTED_EMAIL]`), phone numbers (`[REDACTED_PHONE]`), and SSNs (`[REDACTED_SSN]`) before the query reaches any LLM | Prevents personally identifiable data from being embedded in LLM prompts or audit logs |
| **Prompt Injection Detection** | 6 keyword patterns checked: `"ignore previous instructions"`, `"system prompt"`, `"override instructions"`, `"bypass safeguards"`, `"ignore rules"`, `"you are now a"` | Stops adversarial inputs from hijacking agent behaviour |
| **Domain-Specific Rule** | Queries containing `password`, `credential`, `secret_key`, or `api_key` are blocked | Prevents credential exfiltration via the reporting pipeline |
| **Structured Audit Log** | Every request logged as JSON: `{"severity": "INFO|WARNING|CRITICAL", "audit": {...}}` | Full traceability of security decisions for compliance review |
| **Route-based Blocking** | Blocked requests route to `security_violation_node` вҖ” no further LLM calls made | Ensures zero LLM compute is spent on malicious inputs |

## 5. MCP Server Design

**File:** [`app/mcp_server.py`](app/mcp_server.py)

| Tool | Purpose | Used By |
|------|---------|---------|
| `query_database(query)` | Queries mock corporate database for sales, HR, or customer KPI data | `data_collector` |
| `read_business_file(file_name)` | Reads business logs/CSVs (support tickets, marketing campaigns) | `data_collector` |
| `generate_chart_data(metric_name)` | Returns ASCII bar charts for visual embedding in reports | `report_generator` |

The MCP server runs as a subprocess via stdio transport using `uv run python -m app.mcp_server`. This means no additional port or installation is required вҖ” it starts on-demand when any agent session begins.

## 6. HITL Flow

Human approval is required before any report is dispatched.

| Point | Mechanism | Why |
|-------|-----------|-----|
| After orchestrator generates draft | `RequestInput(interrupt_id="approve_report")` pauses the workflow | Ensures a human reviews content for accuracy before it reaches stakeholders |
| User replies `yes` | Routes to `send_email_node` вҶ’ confirmation shown | Explicit consent to send |
| User replies `no` (or anything else) | Routes to `reject_node` вҶ’ cancellation shown | Prevents accidental or incorrect reports from being sent |

This HITL gate is critical for a business reporting context where incorrect data sent to executives can have significant consequences.

## 8. Impact / Value Statement

ReportGenie demonstrates that AI agents can automate the most repetitive parts of business intelligence work вҖ” data collection, report formatting, and stakeholder communication вҖ” while keeping humans fully in control of what gets sent.

**Who benefits:**
- **Operations & Analytics teams** вҖ” save hours of manual report compilation per week
- **Executives & stakeholders** вҖ” receive consistent, well-formatted reports on demand
- **IT & Compliance teams** вҖ” every request is audited with PII protection built in

**Broader potential:** The architecture is domain-agnostic. Swap the MCP tools for real database connectors, email APIs, or BI systems and the same workflow operates on live enterprise data вҖ” with the same security guarantees and human oversight.
